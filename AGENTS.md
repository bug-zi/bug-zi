# AGENTS.md

## 项目定位

- 这是 `bug-zi/bug-zi` 的 GitHub Profile README 仓库，用来搭建个性化 GitHub 主页。
- 当前主要产物是 `README.md`，GitHub 会直接渲染它作为个人主页。
- 后续开发思路是基于现有内容，参考优秀主页，再按用户明确需求逐步加入个性化模块。

## 工作范围

- 只做用户明确要求的任务，不主动扩展范围。
- 优先编辑现有文件，尤其是 `README.md`。
- 除非用户明确要求，不新增构建系统、脚本、依赖、测试、自动化流程或复杂目录结构。
- 不因为发现相邻问题就顺手修复；需要另行确认。

## 修改原则

- 修改前先阅读相关文件，理解当前结构和内容。
- 使用最小可行改动，避免整文件重写。
- 保持 GitHub Profile README 可直接渲染，不依赖本地构建步骤。
- 新增第三方卡片、徽章、图片或统计组件时，优先使用稳定、常见、适合 GitHub README 的服务。
- 不加入无法在 GitHub README 中正常工作的交互式脚本。

## 内容风格

- 默认使用中文回复用户；README 内容可按现有风格中英混排。
- 文案要简洁、有个人气质，避免堆砌模板化标语。
- 技术栈、项目、博客、联系方式等信息应以用户提供或仓库现有信息为准。
- 参考他人主页时，只借鉴布局和表达方式，不直接复制个人化内容。

## 编码要求

- 所有文本文件默认使用 UTF-8。
- 编辑含中文内容的文件前后，都要注意检查是否出现乱码。
- 在 Windows PowerShell 中查看中文内容时，应设置或考虑 UTF-8 输出编码。
- 如果发现疑似乱码，先区分文件实际损坏和终端显示问题，不要凭终端显示直接判断。

## 非目标

除非用户明确要求，不要执行以下事项：

- 添加测试；
- 添加文档以外的额外文件；
- 重构 README 结构；
- 优化图片资源或外部服务；
- 清理无关内容；
- 修改仓库配置；
- 提交、推送或创建 Pull Request。
